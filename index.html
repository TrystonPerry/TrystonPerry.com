Boarditt

Boarditt is a simple, yet powerful Todo List WebApp. The front-end is built with Angular 5. Using components and services to neatly organize data handled from its back-end API. Most design is built from scratch with vanilla CSS, but also uses Bootstrap 4's grid system. The back-end uses Node.js, Express.js, and MongoDB to form an API for accessing user data like Boards, Lists, and Todos. When logging in or signing up, a user receives a hashed token that is then stored in the cookies to verify they are logged in. This token expires after some time. This token is used to access their personal data across the site.

What is it?

Boarditt is inspired by Googles online todo list app called Google Keep. It allows you to create online sticky notes and lists to organize your tasks in one place. Much like Trello. However, Boarditt features multiple ways to organize your todos more specifically. After signing up, you can create your first board. A board is a project or domain of your life that contains its own todos. Within that board, you can create sub-goals to complete that board. These are called lists. Lists are as they sound, they are a list of individual todos. A todo item is a simple objective for your smaller goal (list). You can check it as done when completing a todo and move onto the others.

Front-End Design

Boarditt uses Angular for the front-end site at Boarditt.com. It features a landing page, login, signup, and home page. The site accesses the private back-end API to create an asynchronous single page app. Since it also features accounts, the design uses cookies to store a temporary login token to access the users' account. Boards, lists, and todos are fetched when loading the dashboard and each is created as its own Angular component. Angular services are used to access API routes across the site.

Back-End Design

Boarditt back-end uses Node.js and Express.js to host a private API. It features GET, POST, PUT, and DELETE request routes for accessing boards, lists, and todos. Board, list, and todo data are stored in a Mongoose model. This mongoose model holds all type of data a board, list, or todo would need. Things like title and color. All user data is stored using a MongoDB database. Almost all requests to the back-end API are required to receive a user access token to verify who is accessing what data. Otherwise, anyone could access anyone's data.

Bubble.js

Open source Javascript library that takes HTML elements and pushes them accross the screen horizontally and vertically. The javascript file can be downloaded manually, cloned, or fetched via a script tag. If you download manually, there are tons of variables to customize bubble position data to your own liking. The github readme also features documentation so a user can easily figure out how the project works.
